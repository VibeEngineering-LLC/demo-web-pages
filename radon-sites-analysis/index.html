Опорные модели

Воронка действия

Check/Test -> Measure -> Act/Protect

UKradon, Take Action - ведёт от вопроса есть ли радон? к измерению и снижению.

Сегментация аудитории

дом / работодатель / профи / регион

UKradon Sector Guidance, Take Action Resources - один контент под разные роли.

Коммерческий слой

продукты + услуги + корзина поверх контента

Radonova, Ecosense, Airthings - монетизация на базе доверия от просветительского ядра.

Митигация-провайдер

услуги + обучение + продукты + онлайн-консультация

American Radon Mitigation, SWAT - подрядчик как контент-эксперт (YouTube/DIY/CE-курсы).

Сайт + лабораторная платформа

публичный сайт <-> система управления измерениями

TPARadon/RNTest: сайт - витрина и вход; протоколы, БД и алгоритмы - во внешней платформе, обмен по API.

Прибор + облако производителя

фирменный монитор -> облако -> приложение/дашборд + подписка

Ecosense (RadonEye), Airthings: наш прибор и облачный сервис строятся по этой же модели; облако прибора - третий контур наряду с сайтом и платформой.

Вовлечение простых потребителей

гражданская наука: измерь сам -> добавь точку на карту -> вовлеки других

Evict Radon, Take Action on Radon, RadonTest (школьники Израиля): население само делает массовые измерения - дешёвый вход (НАРОДНЫЙ метод, QR), своя точка на карте как мотивация, кампании/челленджи, школы и сообщества.

C. Проект RU-сайта: карта разделов (v3)

Костяк - воронка Узнать -> Измерить -> Снизить, дополненная разделами из ТЗ RNTest / Предложения TPARadon: радоновые карты, сервис для профессионалов, личный кабинет. Синяя рамка - контент, жёлтая - pluggable-модули (старт: заглушки скоро), зелёная - разделы, живущие данными из внешней лабораторной платформы.
Ключевое разграничение: web-платформа RNTest/TPARadon из документов - это система управления лабораторией и измерениями. Она НЕ является частью сайта: протоколы методов, базы данных, алгоритмы решений и контроль качества живут в платформе, а сайт - публичная витрина и точка входа. Сайт и платформа обмениваются данными по API. Третий контур - фирменный монитор с облачным сервисом (модель Ecosense/Airthings): прибор шлёт телеметрию в своё облако, облако интегрируется с личным кабинетом сайта и платформой.

  • Обеспечение качества измерений - проверки мониторов и угольного метода в радоновой комнате; прослеживаемость к эталону
  • ★ Онлайн-оценка риска по показаниям прибора ★ УТП - УТП контура: загрузка показаний -> годовая доза + риск

Внешняя лабораторная платформа: что в ней (НЕ на сайте)

  • Протоколы методов измерений: пошаговые workflow народного/угольного/мониторного/детального методов (разделы 2-5 ТЗ), старт/стоп измерений, QR-идентификация образцов
  • Базы данных: СИ (средства измерений), Х2-Х7 (данные методов), ХХ (результаты), Константы, реестр прав доступа
  • Алгоритмы и модули: оценка данных и решение о соответствии нормативу (модули 2.4, 3.3.3.x, 4.3, 7.5), расчёт ОА, поправочные коэффициенты
  • Контроль качества: проверки мониторов и угольного метода (раздел 6 ТЗ), образцовые СИ, метрологическая прослеживаемость
  • Управление лабораторией: регистрация/анализ образцов, интерфейсы детекторов (БДБ-13, МК-2), Комплекс КАМЕРА
  • Пользователи и доступ: учётные записи, права (форма 1.9), абонентские тарифы, чат-претензии (форма 1.8)

Обмен данными сайт <-> платформа

  • Сайт -> платформа: регистрации пользователей и приборов, заявки на услуги/защиту, старт сценариев измерений (переход по ссылке/QR в интерфейс платформы)
  • Платформа -> сайт: результаты и отчёты для личного кабинета, агрегированные данные для радоновых карт, статусы заявок, статистика для публичных страниц
  • Облако прибора <-> платформа/сайт: фирменный монитор шлёт телеметрию в облачный сервис (аналог Ecosense/Airthings Cloud); облако отдаёт данные в личный кабинет на сайте и результаты в платформу - для карт и контроля качества

Движок сайта: WordPress + WooCommerce

  • Контент-разделы (1-3, 5-6, 9-10, 12) -> страницы WP + записи; таксономии для сегментов аудитории (дом/работа/профи/регион) и тем.
  • Раздел 7 Магазин -> WooCommerce Products; часть позиций - витрина маркетплейсов (Яндекс/OZON/WB) с внешними ссылками. Старт: каталог скрыт / скоро.
  • Раздел 8 Услуги -> CPT service (или Woo-booking); форма заявки + калькулятор. Исполнение заявок - workflow во внешней платформе.
  • Разделы 4 и 11 (карты, личный кабинет) - интеграционные: SSO-мост и REST API к платформе; сайт не хранит измерения, только отображает.
  • Онлайн-оценка риска - кастомный шаблон/плагин-виджет (УТП контура).
  • SEO под Яндекс: Schema.org (Product, Organization, FAQPage), ЧПУ-URL; методика seo-audit-ru.
