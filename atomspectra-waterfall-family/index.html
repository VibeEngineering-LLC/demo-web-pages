Зачем это нужно

Четыре рутины, которые обычно съедают время у прибора

Каждый инструмент семейства закрывает одну конкретную проблему полевой и лабораторной работы со спектрометром — не «фичи ради фичей».

Сейчас

Спектрометр физически привязан USB-кабелем к ноутбуку — нельзя отойти, нельзя оставить прибор в поле или в шкафу без ПК рядом.

С шлюзом

Плата размером с ладонь подключается к прибору вместо ПК и раздаёт спектр по WiFi в любой браузер — ноутбук вообще не нужен рядом с прибором.

atomspectra-waterfall-esp32
Сейчас

Запись водопада на много часов — риск: оборвётся WiFi или зависнет скрипт, и часть данных теряется без предупреждения.

С рекордером

Каждый сегмент подтверждается записью на диск (fsync) и проверяется по CRC32 с номером по порядку — обрыв связи не портит уже сохранённые часы записи.

wf-recorder
Сейчас

Суточная запись — десятки тысяч спектров подряд. Искать в ней вспышку активности вручную по таблицам — часы работы.

С вьюером

3D-поверхность Время×Энергия крутится мышью, срез и полоса выделяются прямо на графике, библиотека нуклидов подсвечивает энергии линий — вспышка видна сразу.

waterfall-viewer
Сейчас

Обновление прошивки платы — команды в терминале, риск перепутать COM-порт и вывести из строя чужое USB-устройство.

С флешером

Выбрали проект и порт из списка, нажали «Установить» — актуальная прошивка сама подтягивается с GitHub Releases и заливается по USB.

Как это устроено

Один прибор — четыре точки входа к данным

Плата встаёт между прибором и всем остальным. Дальше данные идут туда, где они нужны: в браузер, в стороннее ПО по TCP или на диск для долгой записи.

Шлюз и флешер взаимодействуют с прибором напрямую. Опция очистки спектра при старте шлюза стирает накопленные на приборе данные — сохраните их штатным ПО заранее. Флешер перезаписывает flash выбранной платы: убедитесь, что порт выбран верно, посторонние USB-устройства прошивать нельзя. Полный текст условий — в README каждого репозитория.
