Stability rule

Точка retry_delay помечается UNSTABLE если выполняется хотя бы одно из:

  • (a) sud_per_min > 0.15 — среднее внезапных разрывов в минуту мониторинга превышает 0.15 (≈9/час)
  • (b) median(records_per_minute) < 17.0

sud_per_min = Σ sudden_disconnects / Σ monitor_time (мин) по всем итерациям точки.
Порог 0.15/мин (≈9/час) физически обоснован: при RSSI −87..−92 dBм (граница чувствительности BLE) редкие единичные разрывы (~1 за 7+ мин) — шумовой пол, не сбой алгоритма. Прежнее правило «≥ 2 итераций с любым разрывом» давало UNSTABLE для всех точек без различия.

MTBF — среднее время между внезапными разрывами

MTBF = Tмонитор / Nsudden для каждой итерации, в минутах. Если N = 0 (разрывов не было) — MTBF = Tмонитор (нижняя граница, отмечена ромбом ◇). Чем выше MTBF — тем стабильнее соединение.

Trecover — среднее время реконнекта

Trecover = Σtreconnect / Nsuccess — среднее время от разрыва до восстановления (сек), только успешные реконнекты. Только итерации с Nsudden > 0.

Psuccess — вероятность успешного восстановления

Psuccess = Nsuccess / Nattempts. 1.0 = каждый разрыв восстановлен. Только итерации с Nsudden > 0.

Score = Trecover / Psuccess — целевая функция (минимизировать)

Объединённый показатель: быстрый и надёжный реконнект → Score мал. Только итерации с обоими значениями.
